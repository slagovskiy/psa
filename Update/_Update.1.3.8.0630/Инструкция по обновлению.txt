1. Обновляется структура базы данных.
	1.1 Запускается робот (желательно последней версии)
	1.2 В настройках отключается галочка импорта с ftp
	1.3 В папку импорта, прописанную в настройках копируются фалы Before*.sql
	1.4 В роботе запускается ручной импорт справочников
	1.5 Пройтись по логу работы робота, в идеале должны происнодить следующий события: 1 - найден файл запроса, 2 - вывод содержимого этого файла в окно робота, 3 - Запрос выполнен успешно. Если что то не выполнилось, скопировать инвормацию об этом и прислать в офис.
	1.6 Известить ответственного за базу клиентов о том, что в  результате обновления все клиенты перешли в категорию розница, пускай подправят базу.
	1.7 Вернуть настройку убранную в 1.2
	1.8 Из папки экспорта удалить все файлы *.info

Примечание: далее справочники категорий клиентов, услуг, дисконтных карт, машин, материалов и прайс выгружается из офиса.

2. Обновляются все файлы проекта, запускается самораспаковывающийся архив, говорите заменить все, при запуске проверяете версию модулей (1.3.8.0624)

3. Обновляется шаблон отчетов. Просто заменяете старый на новый.

4. Появилась возможность сохранять ценрализованно некоторые настройки, но для этого нужно в настройках в разделе администратора указать путь к этому файлу, если у вас есть такая необходимость, то выносите в этот файл интерисующие настройки, если нет, то лучше не выносить, т.к. запрос этих параметров будет выполняться гораздо медленнее чем выборка из реестра, можно, на пример, оставить рекламную информацию в этом файле и префикс заказов.

5. Для обеспечения запуска робота на автомате предлогаю создать на серверах пользователя Robot с паролем robot, зайти под ним, сделать все необходимые настройки, разрешить ему доступ к папке Database (проверить это), что бы он мог писать файлы экспорта. Робот ставится в автозагрузку этого пользователя и убирается у администратора. Запускается файл Autologon.exe, прописываются все параметры для пользователя Robot, жмем Enable, перегружаем сервер и проверяем, что он на автомате залогинится и запускается робот. То же самое нужно проделать с рабочими станциями, где установлено робот (на пример в Молодежке).

Описание изменений смотрите в файле Добавления.txt
Описание выносимых параметров читать там же, они все есть в в примерном файле.
