1. Выбор клиента осуществляется не из всего списка, а из сокращенного, созданного на основе введеных первых символов

2. При добавлении нового клиента его имя вставляется в поле фильтра/поиска и через две секунды он появится в таблице выбора, при этом если было введено уникальное имя, то будет выбран только этот клиент, если нет, то в таблице будет несколь строк

3. В журнале заказа, в шатке таблицы можно задать вильт по номеру заказа, по клиенту и по статусу, при чем в отборе используется любое входение

4. В модуле приемки появилась возможность редактировать справочник клиентов

5. На чеке появилась строка, ограничивающая ответственность. «Внимание: данная стоимость работ (услуг) является приблизительной и может быть увеличена». Обязательно, потребитель должен поставить свою подпись на данном бланке, тем самым он удостоверит свое согласие на превышение стоимости.

6. Список услуг при оформлении заказа раскрашивается в разные цвета, цвет определятеся типом услуги (дизайнер, оператор, обратная) и формой приминения (Количество, кадр, файл, лямда)

7. Запрещено менять окончательную площадь при приеме заказа на лямду, площадь расчитывается автоматически при указании ширины и высоты.

8. При оформлении заказа на Лямду указание имени файла стало обязательным.

9. Убрана ошибка, возникающая в модуле обмена при нажатии на пустом журнале заказов.

10. В модуле обмена по умолчанию фильтр журнала заказов содержит только заказы на выдаче.

11. В модуле администратора добавлен журнал списаний.

12. Администратор может изменить тип списания и того, на кого производится списание. При изменении данне передаются в офис повторно.

13. Добавлена возможность работы с оптовиками "в долг". Если приетн оптовик, то ему можно выдать заказ без оплаты, тогда заказу присваивается статус "Выдано (не оплачено)", потом, в любой момент, заня номер заказа, к нему можно вернуться, внести сумму (если они проходит по наличному расчету) и выставить статус "Выдано".

14. Пересчет стоимости заказа. После того, как заказ будет принят его стоимость расчитывается по фактически выполненному количеству, если какая либо строка не выполнянась вообще, то ее стоимость будет равна нулю.

15. Пересчет стоимости заказа. Одинаковые строки в заказе суммируются и по конечной сумме определяется колонка прайса для получения стоимости заказа.

16. Чек (табличная часть). Списание браков печатается в чеке с минусом в колонке количества.

17. Движение заказа. При возвращении заказа из статуса "безнал" и "предпросмотр" приемке предлагается определить направление движения заказа (дизайнер или оператор) т.к. может быть такая ситуация, что есть еще не выполенные работы дизайнера, а заказа надо передать оператору на печать, то в таком случае автоопределение не сработает.

18. Есть возможность отменять заказа целиком. При это не производится ни какого списания, заказу просто присваивается новый статус с сохранением всех выполенных работ. Если по заказу проходили какие либо деньги, то их необходимо списать введя сумму минусом в кассовый журнал.

19. Дисконтные карты имею свой тип (1,2,3...) в зависимости от этого типа определяются услуги, за которые можно расчитаться бонусами.

20. Перед сохранением заказа выдается запрос на подтвеждение времени выдажи заказа.

21. Если у услуги тип исполнителя равен 3, то заказ, содержащий эту услугу, будет сперва отправлен оператору, потом, если там есть улуги дизайнера, будет отправлен дизайнеру, потом по стандартной схеме.

22. При работе робот использует в качестве кодировки по умолчанию 1251, а не UTF-8

23. Не которые настройки вынесены во внешний файл для централизованного изменения, это DCard limit - Лимит использования карты в течении дня, Dir print - Каталог заказов в печать, Dir edit - Каталог заказов на обработку, Order prefics - префикс номеров заказов, Path Reports Templates - путь к шаблону отчетов, Reklam Block 1 - рекламный блок, Password class 1 - пароль на списания и возвраты. Если параметр не указан в ini файле, то настройки берутся из реестра текущего пользователя. В текущей версии создание т редактирование файла производится вручную.

24. Добавлены новые параметры в настройки: ограничение на использование дисконных карт в течении дня, файл общих настроек.

25. Робот принимает справочники категорий и прайса из офиса, справочники индивидуальные для каждого магазина.

26. Добавлен оговоренный перечень отчетов в модуле администратора

27. Оговорены следующие группы для дисконтных карт: A, B, C, D, E, F, G, H, I, J, K, L, M, а так же группы для купонов и сертификатов: N, O, P, Q, R, S, T, U, V, W, X, Y, Z

28. Появился подробный отчет по кассе, разбит на группы клиентов. Так появися краткий отчет по кассе.

29. Добавлен отчет по деб. задолжности оптовиков, информация о каждом оптовике начинается с новой страницы и содержит информацию о заказах за выбранный период. Отчет во преки ожиданиям считается достаточно быстро, но информация в нем верна только для заказов, которые выданы и в которых не будет менять ни количество оказанных услуг, ни их стоимость.

30. При поиске клиентов в момент обращения к базе выдется информация, что сейчас происходит загрузка.
